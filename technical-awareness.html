Technical Cybersecurity Awareness Matrix
FOCUS AREA DESCRIPTION AWARENESS OBJECTIVE KEY TECHNICAL CONCEPTS PRACTICAL EXAMPLE ATTACKS PREVENTED EXPECTED OUTCOME
Phishing & Email Security Phishing is an attack where attackers send fake emails to trick users into clicking malicious links or entering passwords. Email security protects users from fake emails, malicious attachments, and spoofed domains. Build awareness on email-based compromise and payload delivery. Email header analysis, SPF, DKIM, DMARC, sandboxing, attachment scanning. Analyse a phishing email with spoofed domain and malicious link. Business Email Compromise, Credential Theft, Malware Delivery. Users can identify phishing emails and report them before damage occurs.
Credential & Identity Security This area focuses on protecting user accounts, passwords, and privileged accounts from attackers who try to steal credentials. Prevent compromise through weak or reused credentials. MFA, PAM, password policy, Kerberos/NTLM monitoring. Simulate brute-force login attempts. Account Takeover, Privilege Escalation. Strong authentication and least privilege access.
Patch & Vulnerability Management Attackers exploit software vulnerabilities. Patch management ensures systems are updated and vulnerabilities are fixed. Reduce exposure from known vulnerabilities. CVE awareness, CVSS severity, patch cycle. Scan outdated server and demonstrate exploit. Ransomware, Remote Code Execution. Reduced attack surface.
Endpoint Security Awareness Endpoints like laptops and servers are common attack targets. Monitoring endpoints helps detect malware and suspicious activity. Increase visibility of suspicious endpoint behaviour. EDR telemetry, process injection, persistence registry keys. Analyse abnormal PowerShell activity. Malware, Persistence, Lateral Movement. Early detection of compromised devices.
Network Traffic & Perimeter Awareness/td> This area focuses on monitoring network traffic to detect attackers moving inside networks. Understand how attackers move across network layers. Firewall, IDS/IPS, proxy logging, segmentation. Analyse PCAP showing beaconing traffic. C2 Communication, Lateral Movement. Better network monitoring.
Insider Threat & Data Exfiltration Insider threats occur when employees or internal users steal or leak data intentionally or accidentally. Recognise malicious insider behaviour. UEBA, DLP, file activity monitoring. Detect excessive downloads to USB. Insider Data Theft. Early insider risk detection.
Secure Configuration & Hardening Systems must be securely configured to prevent attackers from exploiting weak settings. Reduce exploitable misconfigurations. CIS benchmarks, disable SMBv1, remove default credentials. Harden Windows server configuration. Privilege Escalation. Secure system configurations.
Cloud Security Awareness Cloud security focuses on protecting cloud storage, IAM roles, and cloud infrastructure. Understand cloud misconfigurations. IAM roles, public storage buckets, CloudTrail logs. Detect public cloud storage exposure. Data Leakage, Account Hijacking. Secure cloud environment.
API & Application Security Awareness APIs and applications must be protected from vulnerabilities and abuse. Prevent logic flaws and data exposure. Authentication, rate limiting, JWT tokens. Detect excessive API calls. API Abuse, Data Exposure. Secure applications.
Incident Response Readiness Incident response ensures organizations can respond quickly to cyber attacks. Ensure teams can respond fast. NIST Incident Response phases. Simulate ransomware incident. Ransomware, Phishing. Faster response time.
Threat Intelligence & IOC Awareness Threat intelligence helps detect attackers using malicious IPs, domains, and file hashes. Enable proactive detection. IOC enrichment, threat feeds. Match malicious IP with threat feed. APT Attacks. Threat-informed detection.
Backup & Ransomware Resilience Backups help organizations recover after ransomware attacks without paying attackers. Ensure backup and recovery readiness. Offline backups, restoration testing. Restore system after ransomware simulation. Ransomware Impact. Business continuity maintained.
Supply Chain & Third-Party Security Supply chain security focuses on risks from vendors, suppliers, and third-party software. If a vendor is compromised, attackers can enter the organization through trusted software or services. Reduce risk from vendors and third-party integrations. Vendor assessment, SBOM, patch assurance, contract security clauses. Analyse software update compromise scenario (SolarWinds-style). Supply Chain Compromise. Third-party risk visibility and contractual security enforcement.
Social Engineering & Human Exploitation Social engineering attacks target humans instead of systems. Attackers trick employees using phone calls, fake emails, or impersonation to steal credentials or sensitive data. Train staff to recognise manipulation attempts. OSINT awareness, vishing simulation, social media policy enforcement. Simulate fake IT support call or pretexting attack. Social Engineering, Account Compromise. Staff verify identities before sharing credentials.
Physical Security & Device Control Physical security protects devices and systems from unauthorized physical access such as USB attacks, stolen laptops, or server room access. Prevent physical access leading to digital compromise. USB control, clean desk policy, asset tagging, CCTV, access logs. Simulate rogue USB insertion or unattended computer access. Physical Intrusion, Data Theft. Reduced insider or onsite compromise risk.
Logging, Monitoring & SIEM Awareness Logging and monitoring help organizations detect attacks early by analysing logs from systems, networks, and applications. Build culture of visibility and continuous detection. Log retention, correlation rules, MITRE ATT&CK mapping, alert triage. Create alert for brute force or DNS tunnelling. Persistent Threats, Reconnaissance. Improved detection maturity and alert quality.
Secure Remote Access & VPN Hygiene Remote access security ensures employees connecting from outside the network use secure VPN connections and strong authentication. Ensure secure external connectivity. TLS encryption, split tunnelling control, device posture check, MFA. Test VPN brute-force and account lockout. VPN Compromise, Credential Abuse. Secure remote access for employees.
Email & Communicat ion Channel Security This focuses on securing communication platforms like email, Teams, Slack, and messaging systems to prevent data leaks and phishing. Secure collaboration platforms. Attachment restriction, link scanning, data loss prevention. Upload sensitive document and trigger DLP alert. Data Leakage, Credential Harvesting. Controlled communication environment.
Continuous Security Education Cybersecurity training must be continuous because attackers constantly change their techniques. Organizations must train employees regularly. Sustain security mindset with evolving threats. Cyber drills, red team awareness, gamified training. Run mini cybersecurity training or CTF. Emerging Threats like AI phishing and deepfake scams. Long-term security culture in organization.

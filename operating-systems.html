Program, Process & Thread

Core
💾
Program

A static file stored on disk. Contains instructions but is not running. Examples: Chrome.exe, python.py

⚙️
Process

A running program. Loaded into RAM, assigned a unique PID, and consumes CPU, memory & I/O.

🧵
Thread

Smallest unit of execution inside a process. Shares the process memory but runs independently.

📌
One-liner to remember

Program is static → Process is running → Thread is a slice of a process.

Process States

Core

A process is never locked in one state. The OS scheduler moves it through a lifecycle based on CPU availability and I/O operations.

🆕

Being created
→
⏳

→
🚀

Using the CPU
→
💤
Waiting
Waiting for I/O
→
✅
Terminated
Execution done

A process in the Waiting state does not consume CPU — it resumes once the I/O operation completes, moving back to Ready.

03

Context Switching

Since only one process can use the CPU at a time, the OS rapidly switches between processes to create the illusion of multitasking. This switch is called a context switch.

⚠️
Performance Cost

Every context switch requires saving and restoring CPU registers, program counters, and memory maps. Too many switches = overhead. The OS minimises unnecessary switches.

Context Switch Flow 1. Save state of Process A (registers, PC, stack pointer)
2. Select next Process B from ready queue
3. Restore state of Process B
4. CPU resumes execution of Process B
04

Memory Management

The OS allocates RAM to processes and enforces memory isolation — no process can read or write another process's memory space (without explicit shared memory).

📚
Stack

Fixed-size. Stores function call frames, local variables. Managed automatically via LIFO. Fast access. Stack overflow if too deep.

🗄️
Heap

Dynamic allocation at runtime via malloc/new. Larger but slower. Developer-managed. Memory leaks occur here.

🔍
Security Note

Buffer overflow attacks exploit stack/heap boundaries. Understanding memory layout is essential for both exploitation and defence.

05

Virtual Memory

Virtual memory extends RAM by using disk space (the swap partition on Linux, pagefile.sys on Windows) as overflow storage.

TermMeaning
PageFixed-size block of virtual memory (typically 4 KB)
FrameCorresponding physical RAM block
PagingMoving pages between RAM and disk
Page FaultAccessing a page not currently in RAM — OS loads it in
ThrashingExcessive paging causing severe performance degradation

User Mode vs Kernel Mode

Security Critical

Modern CPUs provide hardware-enforced privilege rings. The OS uses two primary modes to protect system integrity.

AttributeUser ModeKernel Mode
Hardware AccessRestrictedFull access
Memory AccessOwn process onlyAll memory
Crash ImpactApp crashes onlyEntire system (BSOD / Kernel Panic)
RunsApps, browsers, gamesOS kernel, drivers
Ring LevelRing 3Ring 0
💀
Privilege Escalation

Attackers often aim to move from user mode to kernel mode. Kernel exploits grant total system control — rootkits operate at this level.

08

CPU Scheduling

The scheduler decides which process runs next. Different algorithms optimise for fairness, throughput, or responsiveness.

FCFS
First Come First Serve

Processes are executed in arrival order. Simple but can cause the convoy effect — long jobs block short ones.

RR
Round Robin

Each process gets a fixed time quantum (e.g. 10ms). Fair and responsive. Used in most modern OS schedulers.

PRI
Priority Scheduling

Higher priority processes run first. Risk of starvation — low-priority processes may never execute.

SJF
Shortest Job First

Optimal average wait time but requires knowing process burst times in advance — impractical without estimation.

09

Deadlock

Important

A deadlock occurs when two or more processes are each waiting on a resource held by the other — creating a circular wait with no escape.

Four Coffman Conditions (all must be true for deadlock)

Mutual Exclusion

At least one resource must be held in a non-shareable mode.

Hold & Wait

A process holds resources while waiting to acquire more.

No Preemption

Resources cannot be forcibly taken — only voluntarily released.

Circular Wait

A closed chain of processes where each waits for the next.

🔧
OS Strategies

Prevention — remove one condition.  |  Avoidance — Banker's Algorithm.  |  Detection + Recovery — kill/rollback processes.

Event Viewer Windows

Windows

Event Viewer is Windows' built-in log management tool. Critical for incident response, forensics, and system troubleshooting.

Log CategoryWhat it records
ApplicationSoftware errors, crashes, warnings from installed applications
SystemOS and hardware events — driver failures, service starts/stops
SecurityLogin attempts (success/fail), privilege use, audit policy changes
SetupInstallation and update events

Event Severity Levels

ℹ️
Information

Normal operation recorded.

⚠️
Warning

Non-critical issue that may become a problem.

🔴
Error / Critical

Significant failure requiring attention.

🔍
SOC Analyst Tip

Monitor Event ID 4625 (failed login) and 4672 (special privilege logon). Spikes in these often indicate brute-force or lateral movement.

11

Windows Registry

Windows

The Registry is a hierarchical database storing all Windows configuration — hardware, software, user settings, and system policies. Malware frequently modifies it to achieve persistence.

HKEY_LOCAL_MACHINE System-wide settings, hardware, drivers
HKEY_CURRENT_USER Logged-in user preferences & config
HKEY_CLASSES_ROOT File type associations and COM classes
HKEY_USERS Profiles for all user accounts
🦠
Persistence via Registry

Malware commonly adds entries to HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Run to execute at every boot. Always check this key during forensics.

Linux Permissions

Linux

Linux uses a rwx permission model across three categories: Owner, Group, and Others.

- rwx | r-x | r-- filename.sh
Owner — rwx (7)
Group — r-x (5)
Others — r-- (4)

Key Commands

Permission Commands ls -la # list files with permissions
chmod 755 script.sh # set rwxr-xr-x
chmod +x exploit.sh # add execute bit
chown user:group file # change ownership
find / -perm -4000 # find SUID binaries (privesc)
🎯
SUID / SGID Exploitation

Files with the SUID bit (chmod +s) run with the owner's privileges regardless of who executes them. Finding world-writable SUID binaries is a classic privilege escalation vector.

What is an Operating System? (Beginner Basics)

Beginner

Think of the Operating System as the manager of your computer. Just like a manager in a company handles all the employees and resources — the OS handles all the programs and hardware in your computer.

💡
Simple Real-Life Analogy

Imagine a restaurant. The chef = your application (browser, game). The kitchen manager = the OS. The kitchen equipment = hardware (CPU, RAM, disk). The kitchen manager decides which chef gets to use which equipment and when.

What does the OS actually do?

①
Manages Hardware

Talks to your keyboard, mouse, screen, hard disk on behalf of applications. Without the OS, your game wouldn't know how to use your graphics card.

②
Runs Programs

When you double-click an app, the OS loads it into RAM and gives it CPU time to execute. The app itself can't do this — the OS does it for it.

③
Manages Files

Organises your data into files and folders on the disk. Handles reading, writing, and deleting files safely.

④
Provides Security

Decides which user or program can access what. Prevents one app from reading another app's private data.

⑤
Manages Users

Handles login, multiple user accounts, and permissions — so your files stay private from other users on the same machine.

🖥️
Popular Operating Systems

Windows — most common desktop OS, used in offices worldwide.  |  Linux — used in servers, hacking, and cybersecurity.  |  macOS — Apple computers.  |  Android / iOS — mobile OS.

System Calls — How Apps Talk to the OS

Intermediate

When your program needs to do something that requires hardware (like reading a file, printing to screen, or sending data over the network), it can't do it directly. Instead, it makes a system call — basically asking the OS: "Hey, please do this for me."

📞
Analogy

Imagine you're a customer (app) in a restaurant. You don't go into the kitchen yourself — you call the waiter (system call) who carries your request to the kitchen (OS/kernel) and brings back the result.

Common System Calls

CategorySystem CallWhat it does (simple)
Processfork()Creates a copy of the current process (child process)
Processexec()Replaces current process with a new program
Processexit()Ends the process and frees resources
Fileopen()Opens a file for reading or writing
Fileread()Reads data from a file or input
Filewrite()Writes data to a file or output
Fileclose()Closes a file after you're done with it
Memorymmap()Maps a file or memory into the process address space
Networksocket()Creates a network connection endpoint
🔍
Security Relevance

Malware analysis involves tracing system calls to understand what a suspicious program is doing. Tools like strace (Linux) and Procmon (Windows) capture system calls in real time.

Linux — Trace System Calls strace ls # see all syscalls made by 'ls' command
strace -p 1234 # attach to running process PID 1234
strace -e trace=file ls # show only file-related syscalls

The Booting Process — What Happens When You Press Power

Beginner

Most beginners never think about what happens between pressing the power button and seeing the desktop. Understanding the boot process is critical for cybersecurity — many attacks (like bootkits) target this exact sequence.

🔌
Power ON
CPU wakes up
→
🧩
BIOS / UEFI
Hardware check
→
🔍
POST
Self-test
→
💾
Bootloader
Finds OS
→
⚙️
Kernel
OS loads
→
🖥️
Desktop
Ready to use

Each Step Explained Simply

①
BIOS / UEFI

The first firmware that runs. It lives on a chip on your motherboard. BIOS is old, UEFI is modern. It checks that all hardware (RAM, disk, keyboard) is connected and working.

②
POST (Power-On Self Test)

A quick self-diagnosis of hardware. If something's wrong (e.g., no RAM), you'll hear beeps or see an error — the OS never even loads.

③
Bootloader

A small program that finds the OS on your disk and loads it into RAM. Windows uses bootmgr. Linux uses GRUB. This is where dual-boot menus appear.

④
Kernel Loads

The OS kernel takes over, detects all hardware, loads device drivers, mounts the file system, and starts background services.

⑤
User Space / Desktop

Login screen appears. After logging in, your desktop environment, taskbar, and startup programs launch. The OS is now fully ready.

💀
Bootkit Attacks

Advanced malware called bootkits infect the bootloader or MBR (Master Boot Record) so they load before the OS and antivirus — making them extremely difficult to detect and remove.

Interprocess Communication (IPC)

Intermediate

Processes are isolated from each other (they can't read each other's memory directly). But sometimes they need to share data or coordinate. That's where IPC (Interprocess Communication) comes in.

📬
Real-life Analogy

Two people in separate rooms can't talk directly. They communicate via phone (pipe), shared noticeboard (shared memory), or sending letters (message queue). IPC is the same idea for processes.

IPC MethodHow it works (simple)Speed
Pipe One-way channel — output of one process becomes input of another. Used in Linux terminal with | symbol. Fast
Named Pipe (FIFO) Like a pipe but has a name on the file system — unrelated processes can use it. Fast
Shared Memory Two processes both access the same section of RAM. Fastest method but needs synchronisation. Fastest
Message Queue Processes send messages to a queue; others read from it. Like a mailbox. Medium
Socket Can communicate over a network or locally. Used by browsers, servers, and malware C2. Medium
Signal Simple notification sent to a process — e.g., SIGKILL to terminate, SIGINT when you press Ctrl+C. Instant
IPC — Linux Examples ls | grep ".txt" # pipe: ls output → grep input
kill -9 1234 # send SIGKILL signal to PID 1234
ipcs # list shared memory, queues, semaphores

File System Basics

Beginner

A file system is how the OS organises and stores data on a disk. Without a file system, the disk is just billions of random bits — the file system gives structure, names, and location to everything.

📁
Simple Analogy

Your hard disk is like a giant warehouse. The file system is the shelving system — it labels every shelf (directory), knows where every box (file) is, and records who put it there and when.

Common File Systems

File SystemOSKey Facts
NTFS Windows Supports large files, permissions, encryption (EFS), journaling. Default for Windows.
FAT32 Windows / USB Old, simple, widely compatible. 4 GB max file size limit. No permissions.
exFAT USB / SD Cards Like FAT32 but supports large files. Used on flash drives.
ext4 Linux Default on most Linux distros. Fast, reliable, supports journaling.
XFS Linux High-performance. Used on Red Hat / CentOS servers.
APFS macOS Apple's modern file system. Optimised for SSDs.

Important File System Concepts

🗂️
Inode

A data structure storing file metadata — size, owner, permissions, timestamps. Every file has an inode. The name is stored separately.

📓
Journaling

Keeps a log of changes before writing them. If the system crashes mid-write, the journal allows recovery without data corruption.

🔗
Hard Links

Multiple filenames pointing to the same inode (same data). Deleting one link doesn't delete the data until all links are gone.

Useful Commands df -h # disk space used per file system
du -sh /home/user # size of a specific folder
ls -i # show inode numbers
file secrets.txt # identify true file type
stat file.txt # full metadata — size, timestamps, inode
🔍
Forensics Tip

Even after a file is deleted, the inode and data blocks may still exist on disk until overwritten. Digital forensics tools like Autopsy and Sleuth Kit recover deleted files by scanning for orphaned inodes and data blocks.

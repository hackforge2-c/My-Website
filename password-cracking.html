How Attackers Recover Weak Passwords

Theory

Attackers don't usually try to "break" the encryption itself because that would take far too long with modern cryptography. Instead, they focus on guessing the password that protects the file. The two most common ways of doing this are dictionary attacks and brute-force (or mask) attacks.

🎯 Key Insight: Most passwords are cracked not by breaking cryptography, but by exploiting human weakness — people choose predictable passwords.
02

Dictionary Attacks

Technique

In a dictionary attack, the attacker uses a predefined list of potential passwords, known as a wordlist, and tests each one until the correct password is found.

These wordlists often contain leaked passwords from previous breaches, common substitutions like password123, predictable combinations of names and dates, and other patterns that people frequently use. Because many users choose weak or common passwords, dictionary attacks are usually fast and highly effective.

📌 Famous Wordlist: rockyou.txt — contains 14 million passwords leaked from the RockYou breach in 2009. Pre-installed in Kali Linux at /usr/share/wordlists/rockyou.txt
03

Mask / Brute-Force Attacks

Advanced

Brute-force and mask attacks go one step further. A brute-force attack systematically tries every possible combination of characters until it finds the right one.

Mask attacks aim to reduce that time by limiting guesses to a specific format. For example, trying all combinations of three lowercase letters followed by two digits. By narrowing the search space, mask attacks strike a balance between speed and thoroughness.

Hashcat Mask Characters

Mask Syntax
?l = lowercase letters (a-z)
?u = uppercase letters (A-Z)
?d = digits (0-9)
?s = special characters (!@#$...)
?a = all of the above

Example: ?l?l?l?d?d  = 3 lowercase + 2 digits
         ?u?l?l?l?d  = 1 uppercase + 3 lowercase + 1 digit
04

Practical Tips Attackers Use (and Defenders Should Know)

Intelligence
  • Start with a wordlist (fast wins). Common lists: rockyou.txt, common-passwords.txt.
  • If the wordlist fails, move to targeted wordlists (company names, project names).
  • Try mask attacks on short passwords (example: ?l?l?l?d?d).
  • Use GPU-accelerated cracking when possible.
  • Monitor CPU/GPU usage to detect cracking attempts.
🛡️ Defender Tip: Use passwords longer than 12 characters with mixed case, numbers, and symbols. Enable MFA. Use bcrypt/argon2 for hashing — these are slow by design and make cracking impractical.

Tool Reference

Reference
⚡
John the Ripper
Classic CPU-based hash cracker. Supports hundreds of hash types. Great for ZIP, PDF, shadow files.
🔥
Hashcat
GPU-accelerated cracker — extremely fast. Supports mask attacks, dictionary attacks, rule-based attacks.
📄
pdfcrack
Dedicated PDF password cracker. Simple and effective for dictionary attacks on password-protected PDFs.
🗜️
fcrackzip
ZIP file cracker. Supports both dictionary and brute-force attacks on ZIP archives.
🌊
Hydra
Online password cracker. Attacks login pages, SSH, FTP, and more. Used for credential stuffing.
📝
Crunch
Custom wordlist generator. Create tailored password lists based on known patterns or character sets.

What is Cybersecurity?

Intro

Cybersecurity is the practice of protecting systems, networks, and data from unauthorized access, attacks, or damage. It's essential because today's world is digitally connected — from banking systems to defense infrastructure — and every connection is a potential target.

💡 Why It Matters: With increasing cyber threats, protecting digital assets has become critical for individuals, businesses, and governments worldwide.
02

CIA Triad

Core Concept
The Three Pillars of Information Security

Confidentiality

Ensures data is accessed only by authorized individuals.

Why Confidentiality is Important:

  • Prevents data leaks
  • Protects personal and sensitive information
  • Avoids identity theft and fraud

How Confidentiality is Achieved:

  • Passwords and multi-factor authentication
  • Encryption (AES, RSA)
  • Access control mechanisms
  • Biometrics (fingerprint, face ID)

Integrity

Integrity ensures that data remains accurate, complete, and unchanged unless modified by an authorized person.

Why Integrity is Important:

  • Prevents unauthorized modification
  • Maintains trust in data
  • Ensures data reliability

Real-Life Examples:

  • Exam marks should not be altered
  • Bank balance must remain correct
  • Legal documents must stay original

How Integrity is Maintained:

  • File hashes (SHA-256, MD5)
  • Checksums
  • Digital signatures
  • Version control systems

Availability

Ensures resources are accessible when needed by authorized users.

Real-Life Examples:

  • Banking system working 24/7
  • Company website not crashing
  • Hospital systems available during emergencies

How Availability is Ensured:

  • Regular backups and disaster recovery
  • DDoS protection and mitigation
  • Load balancing and redundancy
  • Failover systems

Threat, Vulnerability & Risk

Understanding the Relationship
TermDefinitionExample
Threat A threat is anything that has the potential to cause harm to systems, data, or individuals. Malware, insider attack, DDoS
Vulnerability A weakness that can be exploited by a threat Unpatched operating system, weak passwords
Risk The probability of loss when a threat exploits a vulnerability Ransomware attack on outdated systems
04

IAAA Framework

Authentication

Identification, Authentication, Authorization, Accounting — The four pillars of access control.

IAAA Process Flow

Identification

Who are you? User claims identity. No verification yet.

  • Username
  • Email ID
  • Employee ID

Authentication

Prove it! Verify the claimed identity.

  • Password
  • OTP (One-Time Password)
  • Fingerprint
  • Face ID
  • Multi-factor authentication (MFA)

Authorization

What can you do? Define permissions and access levels.

  • Student → View marks
  • Teacher → Edit marks
  • Admin → Full access

Accounting (Auditing)

Track everything. Monitor and log all activities.

  • Login time and location
  • Files accessed
  • Changes made
  • Audit trails for compliance

Common Cyber Threats

High Priority
Types of Cyber Threats

Malware

Malicious software designed to harm systems.

  • Virus: Attaches to files and spreads
  • Worm: Self-replicating, spreads without user action
  • Trojan: Disguised as legitimate software
  • Ransomware: Encrypts data and demands payment

Phishing

Fraudulent attempts to obtain sensitive information.

  • Email phishing: Fake emails impersonating trusted sources
  • SMS phishing (Smishing): Via text messages
  • Voice phishing (Vishing): Via phone calls
  • Spear phishing: Targeted attacks on specific individuals

DDoS Attacks

Distributed Denial of Service — overwhelming systems with traffic.

  • Server overload and crashes
  • Website downtime
  • Revenue and reputation loss

Insider Threats

Threats from within the organization.

  • Malicious employees stealing data
  • Careless employees causing breaches
  • Former employees with active credentials

Man-in-the-Middle Attack

Attack
How MITM Attack Works

How MITM Attack Works:

  1. User sends data to the server
  2. Attacker intercepts the data mid-transmission
  3. Attacker can read, modify, or steal the data
  4. Attacker forwards the data to the server
  5. Neither user nor server realizes the breach

Common Types:

  • Wi-Fi eavesdropping: Unsecured public networks
  • Session hijacking: Stealing session cookies
  • ARP spoofing: Redirecting network traffic
  • DNS spoofing: Redirecting to fake websites
  • SSL stripping: Downgrading HTTPS to HTTP

Impact:

  • Password and credential theft
  • Bank fraud and financial loss
  • Identity theft
  • Data manipulation and corruption
🛡️ Prevention Methods:
  • Always use HTTPS websites
  • Avoid public Wi-Fi for sensitive tasks
  • Use VPN (Virtual Private Network)
  • Enable end-to-end encryption
  • Keep systems and software updated
  • Use strong authentication (MFA)

Zero-Day Attacks

Critical
⚠️ Definition: A Zero-Day attack exploits a software vulnerability that is unknown to the software developer and for which no patch or fix exists yet.

Why it is called "Zero-Day":

  • Developers have zero days to fix the vulnerability before it's exploited
  • Attackers exploit it before anyone knows it exists

How it Works:

  1. Unknown vulnerability exists in software
  2. Attacker discovers the flaw (through research or purchase)
  3. Exploits it secretly before disclosure
  4. Damage occurs before developers can release a patch

Why it is Dangerous:

  • No immediate defense available
  • Traditional antivirus may not detect it
  • High success rate for attackers
  • Can cause massive damage to organizations
  • Often sold on dark web markets

Prevention & Mitigation:

  • Apply regular security updates promptly
  • Implement IDS/IPS (Intrusion Detection/Prevention Systems)
  • Use behavior-based security tools
  • Network monitoring and anomaly detection
  • Principle of least-privilege access
  • Security awareness training

Cybersecurity Domains

Network Security

Network Security focuses on protecting computer networks and the data moving through them from unauthorized access, attacks, and misuse.

  • Firewalls and network segmentation
  • IDS/IPS (Intrusion Detection/Prevention Systems)
  • VPN (Virtual Private Networks)
  • Network monitoring and traffic analysis

Application Security

Application Security involves protecting software and applications from vulnerabilities and attacks throughout their lifecycle.

  • SQL Injection prevention
  • XSS (Cross-Site Scripting) protection
  • Broken authentication fixes
  • Secure coding practices
  • OWASP Top 10 compliance

Information Security

Information Security (InfoSec) focuses on protecting data and information from unauthorized access, modification, or destruction.

  • Encryption (at rest and in transit)
  • Access control and permissions
  • Backup and disaster recovery
  • Data classification and handling

Cloud Security

Cloud Security refers to protecting data, applications, and services stored in cloud environments.

  • IAM (Identity and Access Management)
  • Data encryption in the cloud
  • Secure cloud configurations
  • Continuous monitoring and compliance
  • Shared responsibility model understanding

Endpoint Security

Endpoint Security protects end-user devices that connect to a network.

  • Antivirus and anti-malware
  • Device encryption
  • Regular security updates
  • EDR (Endpoint Detection and Response)
  • Mobile device management (MDM)
09

Information Assurance

Information Assurance ensures information is trustworthy throughout its entire lifecycle — from creation to deletion.

Core Activities:

  • Protecting data integrity and confidentiality
  • Managing and mitigating risks
  • Ensuring system reliability and availability
  • Compliance with regulations and standards

IA Includes:

  • Confidentiality: Data accessible only to authorized users
  • Integrity: Data remains accurate and unaltered
  • Availability: Systems and data accessible when needed
  • Authentication: Verifying user identities
  • Non-repudiation: Preventing denial of actions

Access Control Models

Security

Types of Access Control:

  • Mandatory Access Control (MAC): System-enforced, based on security labels
  • Discretionary Access Control (DAC): Owner decides who gets access
  • Role-Based Access Control (RBAC): Access based on user roles
  • Attribute-Based Access Control (ABAC): Dynamic access based on attributes

Access Control Components:

  • Security policies and procedures
  • Access control rules and lists
  • Permission levels (read, write, execute)
  • Access enforcement mechanisms
11

Risk Management

Important
Formula: Risk = Threat × Vulnerability × Impact

Risk Management Process:

  1. Identify assets: Know what you need to protect
  2. Identify threats: What could go wrong?
  3. Identify vulnerabilities: Where are the weaknesses?
  4. Assess risk: Likelihood × Impact
  5. Apply controls: Mitigate, transfer, accept, or avoid
  6. Monitor continuously: Risks evolve over time

Risk Treatment Options:

  • Avoid: Eliminate the risk entirely
  • Mitigate: Reduce likelihood or impact
  • Transfer: Shift risk to third party (insurance)
  • Accept: Acknowledge and accept the risk

Privacy & Data Protection

Personal Data Examples:

  • Name, phone number, email
  • Aadhaar / SSN / National ID
  • Location and tracking data
  • Biometric data (fingerprints, face scan)
  • Financial information
  • Health records

Privacy Principles:

  • Data minimization: Collect only what's necessary
  • Consent: Users must agree to data collection
  • Transparency: Clear communication about data use
  • Purpose limitation: Use data only for stated purposes
  • Data protection: Secure storage and transmission
  • Right to deletion: Users can request data removal
📋 Regulations: GDPR (Europe), CCPA (California), PDPA (Singapore), DPDP Act (India)

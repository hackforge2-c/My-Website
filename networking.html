Networking Fundamentals

Intro

What happens in networking:

  • Data is broken into small packets
  • Each packet contains:
    • Source address (your IP)
    • Destination address (server IP)
  • Data travels through:
    • Router
    • Switch
    • Internet (many networks)
  • Reaches destination and reassembles
📦 How data is broken into packets and encapsulated for transmission
02

Types of Networks

1. LAN (Local Area Network)

  • Small area (home, office, college)
  • High speed
  • Low latency
  • Example: Home WiFi, College lab

2. WAN (Wide Area Network)

  • Covers large area (countries/continents)
  • Internet is the biggest WAN
  • Example: Google servers worldwide

3. MAN (Metropolitan Area Network)

  • Covers a city
  • Example: City-wide ISP

4. PAN (Personal Area Network)

  • Very small range
  • Example: Bluetooth, Mobile hotspot
🌐 PAN → LAN → MAN → WAN — network types by geographic scope
03

Key Performance Metrics

Bandwidth

  • Maximum data that can be sent
  • Measured in Mbps/Gbps
  • Example: 100 Mbps connection

Latency

  • Time taken for data to travel
  • Measured in ms
  • Example: Gaming lag

Throughput

  • Actual speed you get
  • Always less than bandwidth
  • Example: 100 Mbps → 70 Mbps

Jitter

  • Variation in delay
  • Example: Video call lag
04

Network Topology

1. Star Topology

  • All devices connect to central device
  • Pros: Easy to manage, troubleshoot
  • Cons: If central device fails → network fails

2. Mesh Topology

  • Every device connects to every other device
  • Pros: Very reliable, no single failure
  • Cons: Expensive, complex

3. Bus Topology

  • All devices share one cable
  • Pros: Cheap
  • Cons: Cable failure → full network down

4. Ring Topology

  • Devices connected in circle
  • Pros: Data flows in one direction
  • Cons: One failure affects all
🔗 Star, Mesh, Bus and Ring topologies side by side
05

Network Models

1. Client-Server Model

  • Client requests service
  • Server provides service
  • Centralized communication
  • Examples: Websites, Email, Banking apps
  • Security risks:
    • DDoS
    • SQL Injection
    • Unauthorized access
  • If server fails → entire system affected

2. Peer-to-Peer (P2P)

  • No central server
  • Each device acts as client + server
  • Decentralized communication
  • Examples: BitTorrent, LAN sharing, Blockchain

OSI Model

7 Layers

The OSI Model is a 7-layer conceptual framework that explains how data moves from one device to another over a network.

It was standardized by the International Organization for Standardization (ISO) to help vendors and engineers design, build, and troubleshoot network systems in a consistent way.

🧱 OSI Model — all 7 layers from Physical to Application

OSI Layers (Layer 7 → Layer 1):

Layer 1 – Physical Layer:

Transfers raw bits (0s and 1s) over physical media.

Data Unit: Bits

Transmission Media

  • Wired:
    • Coaxial Cable
    • Fiber Optic Cable
  • Wireless:
    • Wi-Fi (Radio Waves)
    • Bluetooth
    • Infrared

Physical layer transmits raw bits over physical media.

Layer 2 – Data Link Layer:

Handles framing, MAC addressing, and error detection.

Data Unit: Frames

Protocols / Standards:

  • Ethernet
  • ARP
  • PPP

Devices:

  • Switches
  • Network Interface Card (NIC)

Data Link layer provides framing, MAC addressing, and error detection.

Layer 3 – Network Layer:

Handles logical addressing and routing between networks.

Data Unit: Packets

Protocols:

  • IP (IPv4 / IPv6)
  • ICMP
  • Routing protocols (OSPF, RIP, BGP)

Devices:

  • Routers
  • Layer 3 switches

Network layer is responsible for logical addressing and routing.

Layer 4 – Transport Layer:

Provides end-to-end communication, flow control, segmentation, and error handling.

Data Unit: Segments

Protocols:

  • TCP (Reliable, ordered, error-checked)
  • UDP (Fast, unreliable)

Transport layer provides reliable or unreliable data delivery using TCP or UDP.

Layer 5 – Session Layer:

  • Session setup & teardown
  • Checkpointing
  • Session recovery

Session layer establishes, manages, and terminates communication sessions.

Layer 6 – Presentation Layer:

  • Data translation (ASCII ↔ Binary)
  • Encryption / Decryption
  • Compression

Protocols / Formats:

  • SSL / TLS
  • JPEG, PNG
  • MP3, MP4

Presentation layer ensures data is properly formatted, encrypted, and compressed.

Layer 7 – Application Layer:

  • HTTP / HTTPS
  • FTP
  • SMTP
  • DNS
  • SNMP

Example:

  • Opening a website in a browser
  • Sending an email

Application layer provides network services directly to end-user applications.

TCP/IP Model

4 Layers

The TCP/IP (Transmission Control Protocol / Internet Protocol) model is a practical and implementable communication model that defines how data is actually transmitted across networks, and it is the foundation of the Internet.

🔄 TCP/IP — the real-world 4-layer model powering the Internet

TCP/IP HAS 4 LAYERS:

  • Application
  • Transport
  • Internet
  • Network Access

1. Application Layer (TCP/IP):

Provides services to user applications like web browsing, email, and file transfer.

Protocols:

  • HTTP / HTTPS
  • FTP
  • SMTP
  • POP3 / IMAP
  • DNS
  • SNMP

Application layer provides network services directly to end-user applications.

2. Transport Layer (TCP/IP):

Provides end-to-end communication, flow control, segmentation, and error handling.

Data Unit: Segment

Protocols:

  • TCP – Reliable, ordered, error-checked
  • UDP – Fast, connectionless, no guarantee

Transport layer ensures reliable or fast data delivery using TCP or UDP.

3. Internet Layer (TCP/IP):

Handles logical addressing and routing of packets.

Data Unit: Packet

Protocols:

  • IP (IPv4 / IPv6)
  • ICMP
  • IPsec

Internet layer handles logical addressing and routing using IP.

4. Network Access Layer (TCP/IP):

Defines how data is physically transmitted, including framing, MAC addressing, and signals.

Data Units: Frame → Bits

Technologies / Media:

  • Ethernet
  • ARP
  • Wi-Fi
  • Cables (UTP, Fiber)
  • MAC Address

Network Access layer handles physical transmission and MAC addressing.

08

Network Devices

🖧 Real-world network setup — router, switch, modem connected together

1. Router

A router connects one network to another network, for example your home network to the internet. It decides where the data should go using IP addresses. A router is like a traffic police that directs data to the correct network.

Cybersecurity Risk:

  • Default username and password can be exploited
  • DNS settings can be changed for phishing attacks
  • Open ports allow attackers to enter the network
  • Remote management can give full control to attacker
  • Outdated firmware can be exploited

Security Tips:

  • Change default password
  • Disable remote access
  • Update firmware
  • Enable firewall
  • Close unused ports

2. Switch

A switch connects devices within the same network like computers, printers, and servers. It sends data only to the correct device using MAC address. A switch is like a post office that delivers letters to the correct person.

Cybersecurity Risk:

  • MAC flooding attack
  • VLAN hopping
  • ARP spoofing
  • Packet sniffing
  • Man-in-the-Middle attack

Security Tips:

  • Enable port security
  • Disable unused ports
  • Use VLANs
  • Enable DHCP snooping
  • Enable Dynamic ARP Inspection

3. Hub

A hub is a basic network device that sends data to all devices in the network. It does not check destination address. It is like a teacher announcing something to the whole class instead of telling one student.

Cybersecurity Risk:

  • Easy packet sniffing
  • No security
  • Easy Man-in-the-Middle attacks

4. Modem

A modem connects your network to the Internet Service Provider (ISP). It converts digital signals to analog and analog to digital. Modem means Modulator + Demodulator.

Cybersecurity Risk:

  • Default password exploitation
  • Remote access attacks
  • DNS hijacking
  • Firmware vulnerabilities

Security Tips:

  • Change modem password
  • Disable remote access
  • Update firmware

5. Access Point

An access point provides Wi-Fi connection to devices and connects them to the wired network. It is like a Wi-Fi signal tower.

Cybersecurity Risk:

  • Rogue Access Point
  • Evil Twin Attack
  • Wi-Fi password cracking
  • Packet sniffing
  • MITM attacks

Security Tips:

  • Use WPA2 or WPA3 encryption
  • Use strong Wi-Fi password
  • Disable WPS
  • Use network segmentation

6. IDS and IPS

IDS (Intrusion Detection System) detects attacks and alerts the administrator. IPS (Intrusion Prevention System) detects and blocks attacks automatically.

VPN

Security

A VPN is a secure communication technology that creates an encrypted tunnel over a public network (like the Internet) to securely transmit data between users and networks.

A VPN is a secure tunnel on the Internet that keeps your data private and hidden.

🔒 VPN creates an encrypted tunnel between your device and the internet

What VPN Does

  • Encrypts data
  • Hides IP address
  • Secures communication over public networks
  • Provides remote access

VPN Protocol Examples

  • IPsec
  • SSL / TLS
  • L2TP
  • OpenVPN
  • WireGuard

A VPN securely encrypts data and provides private communication over public networks.

Firewall

Defence

A Firewall is a network security device or software that monitors, filters, and controls incoming and outgoing network traffic based on predefined security rules.

Firewall can also show configuration details and system logs.

🛡️ Firewall sits between your internal network and the internet — blocking malicious traffic

What Firewall Does:

  • Allows trusted traffic
  • Blocks malicious traffic
  • Enforces security policies
  • Protects internal network

How Firewall Works:

Traffic → Firewall Rules → Allow / Block

Firewalls are used by both network and security teams:

Network Team:

  • Troubleshoots connectivity and network issues
  • Checks allowed/blocked traffic
  • They Will trouble shoot the issues within the internet.

Security Team:

  • Monitors traffic for threats and attacks
  • Analyzes logs, alerts, and incidents.
  • We can monitor the traffic across the organization network.

TYPES OF FIREWALLS:

Host-Based Firewall

  • Installed on individual devices (like laptops, servers, desktops).
  • Controls incoming & outgoing packets for that single host.
  • Useful for protecting devices inside the network (where a network firewall can't see everything).
  • Example: Windows Defender Firewall.

Network-Based Firewall

  • Installed at the network boundary (router, dedicated firewall device).
  • Filters all traffic entering or leaving the entire network.
  • Protects multiple devices inside the internal network.
  • Example: Cisco ASA, Palo Alto Networks firewall.

Simple Formula:

  • Host-based firewall = Guard standing at each house's door.
  • Network-based firewall = Guard standing at the main gate of the whole colony.

Firewall Types

IDS and IPS

Detection

IDS - Intrusion Detection System

IDS monitors network traffic and detects suspicious or malicious activity, then raises alerts.

Key Points:

  • Works like a security camera.
  • Only detects and alerts when it sees suspicious or malicious activity and also detects the vernurability and gives an alert.
  • It does not block the attack — just reports it.

Example:

Multiple wrong login attempts → Alert sent to admin

Formula: IDS = Detects & Reports

IPS – Intrusion Prevention System:

IPS detects malicious activity and takes action to stop it.

Key Points:

  • Works like a security guard.
  • Not only detects but also takes action (blocks, drops, or stops the malicious traffic).
  • Prevents the attack from reaching the system.

Example:

Multiple wrong logins → IP blocked automatically

Formula: IPS = Detects & Blocks

Types of Detection in IDS:

Signature-Based Detection:

  • They are already known activities OR known patterns within the internet.
  • Compares traffic against a internet or database of known attack patterns (signatures).
  • Good at detecting known threats, but cannot detect new/unknown attacks.
  • Example: Detecting a virus because its signature is already in the database.

Anomaly-Based Detection:

  • They are not known or this patterns are not in the internet.
  • Builds a baseline of normal behavior (traffic, user actions, etc.) and raises alerts when something unusual happens.
  • Can detect new, unknown, or zero-day attacks.
  • But may sometimes raise false alarms (false positives).

IP Addressing

Internet Protocol (IP) Address:

An IP address is a unique numerical identifier assigned to a device on a network to enable communication.

An IP address uniquely identifies a device on a network.

Types of IP Addresses:

Private IP Address:

  • Used inside a local or private network.
  • Not routable on the Internet.
  • Not globally unique.
  • Can be reused in different private networks.

Private IP Address Ranges:

ClassRange
Class A10.0.0.0 – 10.255.255.255
Class B172.16.0.0 – 172.31.255.255
Class C192.168.0.0 – 192.168.255.255

Classful addressing exists historically, but CIDR is used today.

Check Your Private IP Address:

  • Windows → ipconfig
  • Linux / macOS → ifconfig or ip addr

Public IP Address:

  • Used on the Internet.
  • Globally unique.
  • Assigned by an Internet Service Provider (ISP).
  • Identifies your network to the outside world.

Check Your Public IP Address:

Search in browser: whatismyip.com or what is my IP

IPv4 vs IPv6

IPv4 (Internet Protocol Version 4)

IPv4 is the most commonly used IP addressing system.

🔢 IPv4 address — 32-bit, four octets separated by dots (e.g. 192.168.1.1)

Structure

IPv4 is 32-bit and written in 4 numbers separated by dots.

192.168.1.1

Each number ranges from 0 to 255.

IPv4 address has two parts:

  • Network part
  • Host part
192.168.1.10
Network = 192.168.1
Host = 10

Problem with IPv4

IPv4 supports about 4.3 billion addresses, and due to many devices (phones, laptops, IoT), IPv4 addresses are running out.

IPv6 (Internet Protocol Version 6)

IPv6 was created to solve IPv4 address shortage.

IPv6 is 128-bit and written in hexadecimal.

2001:0db8:85a3:0000:0000:8a2e:0370:7334

IPv6 allows almost unlimited IP addresses.

Advantages of IPv6

  • More addresses
  • Better security
  • No NAT required
  • Better routing
  • More efficient

IPv4 vs IPv6 Table

FeatureIPv4IPv6
Size32 bit128 bit

Subnetting and CIDR Notation

What is Subnetting?

Subnetting means dividing one large network into smaller networks.

🔀 Subnetting — splitting one large network into smaller manageable sub-networks

Why Subnetting?

  • Better network management
  • Improve security
  • Reduce network traffic
  • Improve performance
  • Network segmentation

Subnet Mask

Subnet mask tells which part is network and which part is host.

IP Address = 192.168.1.10
Subnet Mask = 255.255.255.0

CIDR Notation

CIDR notation is written like:

192.168.1.0/24

/24 means first 24 bits are network bits.

Default Gateway

What is Default Gateway?

Default gateway is the router IP address that devices use to communicate with other networks or the internet.

🚪 Default gateway — the router acts as the exit door from your local network to the internet

Example

DeviceIP
Laptop192.168.1.10
Phone192.168.1.11
Router192.168.1.1

Default Gateway = 192.168.1.1

How it works

  1. Laptop sends data to router
  2. Router sends data to internet
  3. Internet sends response to router
  4. Router sends back to laptop

So router acts as gateway to outside network.

Secure vs Insecure Protocols

Security

What are Secure Protocols?

Secure protocols encrypt data before sending, so attackers cannot read the data.

They provide:

  • Encryption
  • Authentication
  • Data integrity
  • Confidentiality

What are Insecure Protocols?

Insecure protocols send data in plain text, so attackers can capture data using packet sniffing.

Attackers can see:

  • Username
  • Password
  • Emails
  • Data

Common Network Attacks

High Priority

1. Man-In-The-Middle (MITM) Attack

What is MITM?

Man-In-The-Middle attack happens when a hacker secretly stands between two communicating systems and intercepts the communication without their knowledge.

⚠️ MITM — attacker secretly intercepts traffic between victim and server

You → talking to Bank Website
Hacker → standing in the middle
You think you are talking to bank
Bank thinks it is talking to you
But hacker is reading/modifying everything

Real Life Example

When you connect to public Wi-Fi in a coffee shop, an attacker can intercept your traffic and steal:

  • Passwords
  • Emails
  • Credit card details

How attack works (steps)

  1. Attacker connects to same network
  2. Attacker intercepts traffic
  3. Attacker captures or modifies data
  4. User and server don't know attacker exists

Prevention

  • Use HTTPS websites
  • Use VPN
  • Avoid public Wi-Fi
  • Use certificate-based authentication

2. Denial of Service (DoS) Attack

What is DoS?

Denial of Service attack means attacker makes a website or server unavailable by sending too many requests.

The server becomes overloaded and crashes.

Goal of DoS

  • Crash website
  • Stop service
  • Financial loss
  • Reputation damage

Prevention

  • Firewall
  • Rate limiting
  • Load balancer
  • IDS/IPS

3. Distributed Denial of Service (DDoS)

What is DDoS?

DDoS is same as DoS but attack comes from many computers instead of one.

These computers are called botnet (infected systems controlled by attacker).

💥 DDoS — thousands of infected machines (botnet) flood a single target server

Difference between DoS and DDoS

DoSDDoS
One system attacksMany systems attack
Easy to stopHard to stop
Less trafficMassive traffic

Prevention

  • Traffic filtering
  • CDN
  • Anti-DDoS protection
  • Load balancing

4. Packet Sniffing

What is Packet Sniffing?

Packet sniffing means capturing network packets traveling through a network.

🦈 Wireshark — the most popular packet sniffing tool used by attackers and defenders

Packets contain:

  • Username
  • Password
  • Emails
  • Files
  • Network data

If data is not encrypted, attacker can read everything.

Tools used

  • Wireshark
  • tcpdump
  • Ettercap

Prevention

  • Use encryption
  • Use HTTPS
  • Use VPN
  • Use secure protocols (SSH instead of Telnet)

5. IP Spoofing

What is IP Spoofing?

IP Spoofing means attacker changes their IP address to look like a trusted system.

Server thinks:
Request coming from trusted system
But actually attacker sent it.

Why attackers use IP Spoofing?

  • Bypass firewall
  • Perform DDoS
  • Perform MITM attack
  • Hide identity

Prevention

  • Packet filtering
  • Authentication
  • Network monitoring
  • IDS/IPS

6. Port Scanning

What is Port Scanning?

Port scanning is used to find open ports on a system.

🔍 Nmap — the industry-standard port scanner used to discover open services

Every service runs on a port:

  • HTTP → 80
  • HTTPS → 443
  • FTP → 21
  • SSH → 22

Attacker scans ports to find:

  • Open ports
  • Running services
  • Vulnerabilities

Tool

  • Nmap

Prevention

  • Close unused ports
  • Firewall
  • IDS alerts
  • Port filtering

7. Session Hijacking

What is Session Hijacking?

When you login to a website, server gives a session ID.

If attacker steals that session ID, they can login without password.

Example

  1. User logs into website
  2. Server gives session cookie
  3. Attacker steals cookie
  4. Attacker becomes logged-in user

Prevention

  • Use HTTPS
  • Secure cookies
  • Session timeout
  • Multi-factor authentication
